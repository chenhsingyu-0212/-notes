Chapter 2_Network Models


2.1 Protocol Layering

Scenarios

  • First Scenario:
    在第一種情況下，通信非常簡單，只能在一層中進行。



  • Second Scenario:
    通過帖子使用普通郵件對話。但是，如果信件被截獲，不希望自己的想法被其他人透露。
    用加密/解密技術，信件的寄件者對其進行加密，使其無法被入侵者讀取。信件的接收者對其進行解密以獲取原始信件。



Principles of Protocol Layering

  • First Principle:
    想要雙向通信，我們需要使每一層都能夠執行兩個相反的任務。

  • Second Principle:
    兩個網站上每層下的兩個對象應該是相同的。都是文字或是都是密文。

Logical Connections

每一層都有一個邏輯（虛構的）連接，通過它他們可以發送從該層創建的物件。
邏輯連接的概念將幫助我們更好地理解我們在數據通信和網路中遇到的分層任務。




2.2 TCP/IP Protocol Suite

TCP/IP (Transmission Control Protocol/Internet Protocol)。
TCP / IP是當今互聯網中使用的協定套件(一組組織在不同層中的協定)。
它是一個由互動式模組組成的分層協定，每個模組都提供特定的功能。
術語「分層」是指每個上層協定都由一個或多個較低級別協定提供的服務支援。

Layered Architecture




  • 兩個主機涉及所有五個層:
    源主機需要在應用程式層中創建一條消息，並將其發送到各層，以便將其物理發送到目標主機。
    目標主機需要在物理層接收通信，然後通過其他層將其傳遞到應用程式層。

  • 路由器僅涉及三層:
    只要路由器僅用於路由，路由器中就沒有傳輸層或應用層。
    儘管路由器始終涉及一個網路層，但它涉及 n 個鏈路層和物理層的組合，其中 n 是路由器連接到的鏈路數。

  • 鏈路中的鏈路層交換機僅涉及兩層：
    數據鏈路和物理層。
    儘管每個交換機都有兩個不同的連接，但這些連接位於同一鏈路中，該鏈路僅使用一組協定。
    這意味著，與路由器不同，鏈路層交換機僅涉及一個數據鏈路和一個物理層。

Layers in the TCP/IP Protocol Suite



應用程式、傳輸層和網路層的職責是端到端的(end-to-end)。
但是，數據鏈路和物理層的職責是跳到跳(hop-to-hop)，其中躍點是主機或路由器。
換句話說，前三層的責任域是互聯網，而兩層下層的責任域是連結。
任何路由器或鏈路層交換機都不應更改數據單元（數據包）。
在最下面的兩層中，主機創建的數據包僅由路由器更改，而不由鏈路層交換機更改。



儘管網路層的邏輯連接位於兩個主機之間，但在這種情況下，我們只能說兩個躍點之間存在相同的對象，因為路由器可能會在網路層對數據包進行分段，併發送比接收的更多的數據包。
請注意，兩個躍點之間的連結不會更改物件。

Description of Each Layer

  • Physical Layer:
    定義一些電器，機械，過程和規範。

  • Data-link Layer:
    主要是用來定義來源/目的的MAC地址，以及所使用的協定為何(IPV4/IPV6)。
    它是封包從一個裝置的網路層傳輸到另外一個裝置的網路層的方法。

  • Network Layer:
    網路層解決在一個單一網路上傳輸封包的問題、將資料從源網路傳輸到目的網路，這就牽涉到在網路組成的網上選擇路徑將封包傳輸。
    會在此定義該封包需要送往哪一個IP位置。

  • Transport Layer:
    傳輸層的協定，能夠解決諸如端到端可靠性(「資料是否已經到達目的地？」)和保證資料按照正確的順序到達這樣的問題。
    紀錄來源/目的Port位置，此外亦會給每一個封包追蹤號碼，並在目的地進行檢查。

  • Application Layer:
    該層包括所有和應用程式協同工作，利用基礎網路交換應用程式專用的資料的協定。
    位址用名字表示，如:@gmail.com

Encapsulation and Decapsulation



  • Encapsulation at the Source Host:
    在源頭上，我們只有封裝。
    1. 在應用程式層，要交換的數據稱為消息(message)。
      消息通常不包含任何標頭或尾部。
      消息將傳遞到傳輸層。

    2. 傳輸層將消息作為有效負載，即傳輸層應處理的負載。
      它將傳輸層標頭添加到有效負載中，其中包含要通信的源和目標應用程式識別碼，以及端到端傳遞消息所需的一些更多資訊，例如流、錯誤控制或擁塞控制所需的資訊。 結果是傳輸層數據包，稱為段(segment，在 TCP 中)和用戶數據報(user datagram，在 UDP 中)。
      然後，傳輸層將數據包傳遞到網路層。

    3. 網路層將傳輸層數據包作為數據或有效負載，並將其自己的標頭添加到有效負載中。
      標頭包含源主機和目標主機的位址，以及用於標頭錯誤檢查、碎片資訊等的一些更多資訊。
      結果是網路層數據包，稱為數據報(datagram)。
      然後，網路層將數據包傳遞到數據鏈路層。

    4. 數據鏈路層將網路層數據包作為數據或有效負載，並添加自己的標頭，其中包含主機或下一躍點（路由器）的鏈路層位址。
      結果是鏈路層數據包，稱為幀(frame)。
      幀被傳遞到物理層進行傳輸。

  • Decapsulation and Encapsulation at the Router:
    在路由器上，我們同時具有解封裝和封裝，因為路由器連接到兩個或多個鏈路。
    1. 將一組位傳遞到數據鏈路層后，此層將從幀中解封數據報並將其傳遞到網路層。

    2. 網路層僅檢查數據報頭中的源位址和目標位址，並查閱其轉發表以查找要將數據報傳遞到的下一躍點。
      路由器中的網路層不應更改數據報的內容，除非如果數據報太大而無法通過下一個鏈路，則需要對數據報進行分段。
      然後，數據報將傳遞到下一個鏈接的數據鏈路層。

    3. 下一個鏈接的數據鏈路層將數據報封裝在幀中，並將其傳遞到物理層進行傳輸。

  • Decapsulation at the Destination Host:
    在目標主機上，每一層僅解封收到的數據包，刪除有效負載(payload)，並將有效負載傳遞到下一個更高層的協定，直到消息到達應用層。
    有必要說主機中的解封涉及錯誤檢查。

Addressing

任何涉及雙方的通信都需要兩個位址：源位址和目標位址。雖然看起來我們需要五對位址，每層一對，但我們通常只有四對，因為物理層不需要位址;物理層的數據交換單位是位，絕對不能有位址。



Multiplexing and Demultiplexing

由於TCP/IP協定套件在某些層使用多個協定，因此我們可以說我們在源位置具有多路複用，在目標位置具有解複用。




2.3 OSI Model

  • International Organization for Standardization (ISO) :是一個致力於全球國際標準協定的跨國機構。
  • Open Systems Interconnection (OSI) model :涵蓋了網路通信的所有方面。
ISO is the organization; OSI is the model.
