Chapter 1_Introduction

1.1 Data Communications

Components

Telecommunication(電話通訊) v.s. Data communication(資料通訊)

  • telecommunication:
    電話接通後，電話兩端不可再有人同時打入電話。

  • data communication:
    達到可以多方傳遞，或只同時傳遞。

Five components of date communication:
  1. Sender(寄送人):
    The sender is the device that sends the data message.

  2. Receiver(接收人):
    The receiver is the device that receives the message.

  3. Message(訊息):
    The message is the information (data) to be communicated.

  4. Transmission medium(傳輸介質):
    The transmission medium is the physical path by which a message travels from sender to receiver.

  5. Protocol(協定):
    A protocol is a set of rules that govern data communications.

Data Flow

Communication between two devices:

  • Simplex:
    The communication is unidirectional.(一個只傳送訊息，另一個只接收)

  • Half-duplex:
    Each station can both transmit and receive, but not at the same time. When one device is sending, the other can only receive.(都可以傳送和接收，但不能兩邊同時做)

  • Full-duplex(duplex):
    Both stations can transmit and receive simultaneously.(兩邊可以同時發送與接收)

1.2 Networks

A network is the interconnection of a set of devices capable of communication.
兩種物體之間，透過某種形式互相連接，以達到傳送的目的。
由軟硬體組成，並將資料從本地端傳送到另一端。

Devices in the definition can be:

  • host(主機)

  • connecting device(連接設備)

    • router(路由器):
      connects the network to other networks.

    • switch(交換機):
      connects devices together.

    • modem(modulator-demodulator)(數據機):
      changes the form of data.


Network Criteria

  1. Performance(性能):
    測量效能有很多方法，包括傳送時間、回覆時間。
    傳送時間是從一端傳送訊息至另一端的時間總和。
    回覆時間是個詢問和反應之間所經過的時間。
    網路效能依賴一些因素，包含使用者的數量、傳輸媒介的種類、連接硬體的能力、軟體的效率。
    Performance is often evaluated by two networking metrics: throughput and delay.

  2. Reliability(可靠性):
    除了交付的準確性之外，網路可靠性還通過故障頻率、從故障中恢復所需的時間、網路在災難中的穩健性來衡量。

  3. Security(安全性):
    網路安全問題包含保護資料免於未經許可的存取、損害與改變，以及實現從資料遺失和破壞中恢復的原則和程序。

Physical Structures

Type of Connection:

  • Point-to-Point:
    A point-to-point connection provides a dedicated link between two devices.

  • Multipoint:
    A multipoint connection is one in which more than two specific devices share a single link.


Physical Topology

  • Mesh Topology(網格拓撲):
    Every device has a dedicated point-to-point link to every other device.
    We need n (n - 1) / 2 duplex-mode links.

    Advantages:
    1. 使用專用鏈路保證每個連接都可以承載自己的數據負載，如果要消除鏈路必須由多個設備共用發生的流量問題。
    2. 網格拓撲是強大的。如果一個連結不可用，它不會使整個系統喪失能力。
    3. 有隱私或安全的優勢。

    disadvantages:
    1. 由於每個設備都必須連接到其他所有設備，因此安裝和重新連接很困難。
    2. 布線的絕對體積會大於可用空間可以容納的範圍。
    3. 連接每個鏈路（I/O 埠和電纜）所需的硬體非常昂貴。



  • Star Topology(星狀拓撲):
    Device has a dedicated point-to-point link only to a central controller, usually called a hub. The devices are not directly linked to one another.
    星形拓撲用於局域網(LAN)。高速 LAN 通常使用帶有中央集線器的星形拓撲。


  • Bus Topology(匯流排拓撲):
    It is multipoint. One long cable acts as a backbone to link all the devices in a network.
    匯流排拓撲是早期局部區域網路設計中使用的首批拓撲之一。傳統的乙太網 LAN 可以使用匯流排拓撲，但它們現在不太受歡迎了。


  • Ring Topology(環形拓撲):
    Device has a dedicated point-to-point connection with only the two devices on either side of it.
    當IBM推出其局域網Token Ring時，環形拓撲很普遍。如今，對更高速 LAN 的需求使得這種拓撲不太受歡迎。


1.3 Network Types

Local Area Network

局域網 （LAN） 通常是私有的，連接單個辦公室、建築物或園區中的某些主機。
LAN 中的每個主機都有一個標識碼和一個位址，該標識碼和位址唯一地定義 LAN 中的主機。
一個主機發送到另一個主機的數據包同時攜帶源主機和目標主機的位址。

過去，網路中的所有主機都通過公共電纜連接，這意味著所有主機都會接收從一台主機發送到另一台主機的數據包。預期的收件者保留了數據包;其他人丟棄了數據包。
如今，大多數 LAN 都使用智慧連接交換機，該交換機能夠識別數據包的目標位址，並將數據包引導到其目標，而無需將其發送到所有其他主機。



Wide Area Network

WAN具有更寬的地理跨度，跨越城鎮、州、國家甚至世界。

  • Point-to-Point WAN
    是透過傳輸介質（電纜或空氣）連接兩個通信設備的網路。


  • Switched WAN
    是具有兩端以上端的網路。
    用於當今全球通信的骨幹網。
    交換 WAN 是通過交換機連接的多個點對點 WAN 的組合。


Internet

今天，很少看到LAN或WAN是孤立的。它們彼此相連。
當連接兩個或多個Network時，它們會建立internetwork或internet。
例子:管理層租用了一個點對點專用WAN從服務提供者（如電話公司）並連接兩個 LAN。
當一台主機向同一辦公室中的另一台主機發送消息時，路由器(router)會阻止該消息，但交換機(switch)會將消息定向到目標。

Switching

An internet is a switched network in which a switch connects at least two links together.

  • Circuit-Switched Network
    在電路交換網路中，兩端系統之間始終可以使用稱為電路的專用連接。交換機只能使其處於活動狀態或非活動狀態。


  • Packet-Switched Network
    在計算機網路中，兩端之間的通信是在稱為數據包(packets)的數據塊中完成的。分組交換網路中的路由器具有可以存儲和轉發包。


The Internet

internet v.s. Internet

  • internet(注意小寫的i):
    是兩個或多個可以相互通信的網路。

  • Internet(大寫I):
    最著名的internet被稱為Internet，由數千個internetwork組成。


Backbones和provider networks也稱為互聯網服務提供者(Internet Service Providers(ISPs))。
Backbones通常被稱為國際互聯網服務提供者;provider networks通常被稱為國家或區域ISP。

1.4 Internet History

Birth of the Internet

1972年，Vint Cerf和Bob Kahn都是ARPANET核心團隊的一員，他們合作開展了他們所謂的互聯網專案。他們希望連結不同的網路，以便一個網路上的主機可以與另一個網路上的主機進行通信。

1.5 Standards and Administration

Internet Standards

互聯網標準(Internet Standards)是一個經過全面測試的規範，對使用互聯網的人有用並遵守。這是一項必須遵守的正式規定。規範達到互聯網標準狀態有一個嚴格的程式。
規範從互聯網草案(Internet draft)開始。互聯網草稿是一份工作檔（一項正在進行的工作），沒有正式地位，有效期為六個月。
根據互聯網當局的建議，草案可以作為徵求意見書(Request for Comment(RFC))發佈。每個 RFC 都經過編輯，分配一個編號，並提供給所有感興趣的各方。RFC 經過成熟度級別(Maturity Levels)，並根據其要求級別(Requirement Levels)進行分類。

  • Maturity Levels:
    建議的標準、草案標準、互聯網標準、歷史標準、實驗性和資訊性。



  • Requirement Levels:
    必需、推薦、選修、有限使用和不推薦

Internet Administration

The general organization of Internet administration(互聯網管理的一般組織)

  • Internet Society (ISOC) :
    為互聯網標準流程提供支援。

  • Internet Architecture Board (IAB) :
    是ISOC的技術顧問。

  • Internet Engineering Task Force (IETF) :
    是由互聯網工程指導小組(IESG)管理的工作組論壇。
    IETF負責識別操作問題並提出這些問題的解決方案。

  • Internet Research Task Force (IRTF) :
    是由互聯網研究指導小組(IRSG)管理的工作組論壇。
    IRTF專注於與互聯網協定，應用程式，架構和技術相關的長期研究課題。
